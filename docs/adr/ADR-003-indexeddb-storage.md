# ADR-003: IndexedDB 本地存储历史记录

**日期**: 2026-05-10
**状态**: ✅ 已接受

## 背景

如何存储用户的点评历史记录？

## 决策

使用 IndexedDB 本地存储。

## 理由

- 无需后端服务（降低复杂度）
- 支持大对象存储（base64 图片）
- 浏览器原生支持，无需额外依赖
- 用户数据完全本地，隐私友好

## 后果

- 数据仅在当前浏览器可用
- IndexedDB 测试在 Node.js 环境下不可用
- 需要处理浏览器兼容性（现代浏览器已支持）

## 替代方案考虑

- localStorage: 容量限制（~5MB），不适合存图片
- 后端数据库: 需要额外基础设施
- IndexedDB: ✅ 最佳平衡
