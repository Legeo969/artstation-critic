### Issue #8: 拖拽上传 + 剪贴板粘贴

**优先级**: P0

**目标**: 支持拖拽图片上传和 Ctrl+V 剪贴板粘贴

**技术方案**:
- Drag & Drop API
- `navigator.clipboard.read()` 剪贴板读取
- 拖拽区域视觉反馈

**预计时间**: 1-2 天

---

### Issue #9: AI 生成对比优化图

**优先级**: P1

**目标**: AI 不仅点评，还能生成优化版本

**技术方案**:
- 使用 GPT-4o（多模态，支持图像生成）
- Prompt: "请基于以下点评，生成优化后的作品"
- 原图 vs 优化图并排对比

**注意**: GPT-4o 需要付费 API Key

**预计时间**: 3-5 天

---

### Issue #10: 批量上传（5 张）

**优先级**: P2

**目标**: 一次上传最多 5 张图片

**技术方案**:
- `<input multiple>` 支持
- 上传队列 UI（进度条）
- 串行处理（逐个上传）
- 单个失败不影响其他

**预计时间**: 2-3 天

---

### Issue #11: 同图多次上传对比

**优先级**: P3

**目标**: 追踪同一作品的改进进度

**技术方案**:
- IndexedDB 按文件名分组
- 折线图展示 6 维度评分趋势
- 选择两个版本对比
- 轻量级图表库（Chart.js 或纯 SVG）

**预计时间**: 2-3 天

---

### Issue #12: UI/UX 优化

**优先级**: P4

**目标**: 提升用户体验，符合主流设计趋势

**内容**:
- 深色模式完善（系统偏好 + 手动切换）
- 响应式布局优化（移动端适配）
- 加载状态（骨架屏、进度条）
- 微交互（hover/active 状态）
- 动画过渡（页面切换）
- 可访问性改进（ARIA、键盘导航）

**参考**: mattpocock/skills 的 prototype skill 原则

**预计时间**: 2-3 天

## 决策原则

1. **用户价值优先**: 优先实现用户最需要的功能
2. **技术可行性**: 优先选择成熟稳定的技术方案
3. **开发效率**: 小步快跑，快速迭代
4. **质量保障**: 每个功能必须有测试
