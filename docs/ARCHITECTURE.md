# 架构设计

## 概述

Artstation Critic 是一个基于 Next.js 16 的 Web 应用，使用 AI 对美术作品进行多维度点评。


## 关键设计决策

### 1. Server Action 用于 AI 调用

**原因**: 
- 保护 API Key 不暴露在客户端
- 减少客户端包体积
- 利用 Next.js 原生 Server Action

### 2. IndexedDB 本地存储

**原因**:
- 无需后端服务
- 支持大对象存储（base64 图片）
- 浏览器原生支持

### 3. 多 API 提供商支持

**原因**:
- DeepSeek（推荐）：免费额度，兼容 OpenAI 格式
- 火山方舟：备选方案
- 通过 Settings 页面动态切换

### 4. 中文界面 + 英文品牌

**原因**:
- 目标用户主要是中文使用者
- 保留 "Artstation Critic" 品牌识别度
- 维度名称双语显示（方便对照）

## 性能考虑

| 场景 | 优化策略 |
|------|----------|
| 图片上传 | 前端压缩（可选） |
| AI 响应 | 进度条 + 加载状态 |
| 历史记录 | IndexedDB 索引优化 |
| 首屏加载 | Next.js Turbopack 热更新 |

## 安全考虑

| 风险 | 缓解措施 |
|------|----------|
| API Key 泄露 | 存储在 Server Action，不暴露给客户端 |
| 图片大小 | 前端限制 + 后端验证 |
| XSS | React 默认转义 + Content Security Policy |
